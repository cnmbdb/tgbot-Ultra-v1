# UI UX Pro Max

An AI skill that provides design intelligence for building professional UI/UX across multiple platforms and frameworks.

## When to Use
Invoke this skill when the user wants to:
- Build a new UI component or page
- Redesign an existing interface
- Improve UX/UI of a specific section
- Generate a design system
- Choose colors, typography, or layout styles

## Core Capabilities

### 1. Intelligent Design System Generation
Analyzes project requirements and generates a complete, tailored design system in seconds, including:
- **Pattern**: Landing page structure, dashboard layout
- **Style**: Visual aesthetic (e.g., Soft UI, Glassmorphism)
- **Colors**: Primary, secondary, CTA, background palettes
- **Typography**: Font pairings and moods
- **Key Effects**: Shadows, transitions, interactions
- **Anti-patterns**: What to avoid for the specific industry

### 2. Industry-Specific Reasoning
Includes specialized rules for:
- Tech & SaaS
- Finance & Fintech
- Healthcare
- E-commerce
- Services & Lifestyle
- Creative & Emerging Tech

### 3. Extensive Style Library
Supports 67+ UI styles including:
- **General**: Minimalism, Neumorphism, Glassmorphism, Brutalism, Dark Mode
- **Landing Pages**: Hero-Centric, Conversion-Optimized, Trust & Authority
- **Dashboards**: Data-Dense, Heat Map, Executive, Real-Time Monitoring

## How to Use

### Step 1: Analyze Context
Determine the product type (e.g., SaaS, Admin Panel, E-commerce) and the user's goal.

### Step 2: Select Style & Pattern
Based on the product type, select the most appropriate style from the library.
*Example for Admin Panel:* "Data-Dense Dashboard" or "Minimalism & Swiss Style".

### Step 3: Generate Design System
Create a design system definition (Colors, Fonts, Spacing, Shadows).

### Step 4: Implementation
Apply the design system to the code.
- **HTML/CSS**: Use Tailwind classes or custom CSS variables.
- **Components**: Update standard components (Buttons, Cards, Inputs, Tables).
- **Layout**: Refactor page layouts to match the selected pattern.

## Anti-Patterns to Avoid
- **General**: Bright neon colors in serious apps, harsh animations, poor contrast.
- **Admin Panels**: Cluttered layouts, low information density where high density is needed, lack of visual hierarchy.
