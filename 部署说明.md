# TGBot-Ultra-v1 服务器部署说明

## 📋 部署前检查清单

### ✅ 必需文件（已包含）
- [x] `docker-compose.yml` - Docker Compose 配置文件
- [x] `admin.env` - Admin 服务环境变量配置
- [x] `job.env` - Job 服务环境变量配置
- [x] `001_init.sql` - 数据库初始化脚本（表结构 + 初始数据 + 视图）
- [x] `DB_PostgreSQL_views_no_prefix.sql` - 兼容视图创建脚本
- [x] `admin/config/permission.php` - 权限配置文件

### ⚙️ 部署前需要修改的配置

#### 1. 修改 `admin.env` 文件
```bash
# 修改以下配置项：
APP_URL=http://your-server-domain:8080  # 改为实际服务器域名或IP
DB_PASSWORD=your-secure-password         # 修改数据库密码（如果使用默认密码，建议修改）
```

#### 2. 修改 `job.env` 文件
```bash
# 修改以下配置项：
DB_PASSWORD=your-secure-password         # 与 admin.env 保持一致
```

#### 3. 修改 `docker-compose.yml`（可选）
```yaml
# 如果需要修改端口，可以通过环境变量设置：
ADMIN_PORT=8080    # Admin 服务端口
JOB_PORT=9503     # Job 服务端口
POSTGRES_PORT=5432 # PostgreSQL 端口
REDIS_PORT=6379   # Redis 端口

# 数据库密码（如果修改了 admin.env 和 job.env 中的密码）
DB_PASSWORD=your-secure-password
```

## 🚀 部署步骤

### 1. 上传文件到服务器
```bash
# 将整个目录上传到服务器，例如：
scp -r /Users/a2333/IDE/部署到服务器的文件/tgbot-Ultra-v1 user@server:/www/wwwroot/
```

### 2. 登录服务器并进入目录
```bash
ssh user@server
cd /www/wwwroot/tgbot-Ultra-v1
```

### 3. 修改配置文件
```bash
# 编辑配置文件，修改数据库密码、域名等
nano admin.env
nano job.env
```

### 4. 拉取镜像并启动服务
```bash
# 拉取最新镜像（支持多架构：amd64/arm64）
docker-compose pull

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 5. 验证部署
```bash
# 检查容器状态
docker-compose ps

# 检查 Admin 服务
curl -I http://localhost:8080

# 检查 Job 服务
curl http://localhost:9503
```

## 📝 重要说明

### 镜像信息
- **Admin 镜像**: `hfdoker2333/tgbot-ultra-v1:admin-latest`
  - 支持架构: `linux/amd64`, `linux/arm64`
  - 已包含 Laravel 必需的目录结构
  
- **Job 镜像**: `hfdoker2333/tgbot-ultra-v1:job-latest`
  - 支持架构: `linux/amd64`, `linux/arm64`

### 数据库初始化
- 首次启动时，PostgreSQL 会自动执行初始化脚本
- `001_init.sql` - 创建所有表结构及视图
- `DB_PostgreSQL_views_no_prefix.sql` - 创建兼容视图（无前缀表名）

### 数据持久化
以下数据会持久化到 Docker volumes：
- `postgres_data` - PostgreSQL 数据库数据
- `admin_storage` - Admin 服务存储数据
- `admin_cache` - Admin 服务缓存
- `job_storage` - Job 服务运行时数据
- `redis_data` - Redis 数据

### 端口说明
- **8080**: Admin 后台管理界面
- **9503**: Job 服务端口
- **5432**: PostgreSQL 数据库（建议不对外开放）
- **6379**: Redis 缓存（建议不对外开放）

## ⚠️ 注意事项

1. **首次部署**：确保数据库密码已修改，不要使用默认密码
2. **HTTPS 配置**：如果使用 HTTPS，需要修改 `admin.env` 中的 `IS_HTTPS=true` 和 `APP_URL`
3. **Webhook 配置**：Telegram Webhook 只支持 80/443/88/8443 端口，需要配置反向代理
4. **防火墙**：确保服务器防火墙开放了必要的端口
5. **备份**：定期备份 `postgres_data` volume 中的数据

## 🆘 故障排查

### 服务无法启动
```bash
# 查看详细日志
docker-compose logs --tail=100

# 检查容器状态
docker-compose ps -a

# 检查端口占用
netstat -tlnp | grep -E '8080|9503|5432|6379'
```

### 数据库连接失败
```bash
# 检查 PostgreSQL 容器
docker-compose logs postgres

# 测试数据库连接
docker exec -it tgbot-postgres psql -U root -d tgbot
```

### Admin 页面无法访问
```bash
# 检查 Admin 容器日志
docker-compose logs admin

# 检查 Nginx 和 PHP-FPM
docker exec tgbot-admin ps aux | grep -E 'nginx|php-fpm'
```
